# AB Unity MCP Project Context

This folder contains concise project context for agents connecting through the
AB Unity MCP plugin. Keep it aligned with the root `AGENTS.md` and the project
docs under `Docs/`.

Rules in this folder are project guidance, not a replacement for source code,
tests, or Unity validation. When rules conflict, prefer the root `AGENTS.md`.

Useful files:

- `ProjectGuidelines.md`: project structure, Git, and asset rules.
- `Architecture.md`: current technical architecture and module map.
- `GameDesign.md`: core game promise, loop, and MVP scope.
- `NetworkingGuidelines.md`: NGO authority and synchronization rules.
- `NetworkingCSP.md`: multiplayer validation and endpoint boundaries.

Do not store passwords, tokens, or machine-local secrets here.
